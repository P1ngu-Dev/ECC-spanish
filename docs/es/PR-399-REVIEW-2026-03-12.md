## Hallazgos

### Medium

1. `skills/continuous-learning-v2/hooks/observe.sh`

La nueva protección `CLAUDE_CODE_ENTRYPOINT` usa una allowlist finita de valores
no-`cli` conocidos (`sdk-ts`, `sdk-py`, `sdk-cli`, `mcp`, `remote`).

Eso deja un hueco de compatibilidad futura: cualquier valor nuevo de entrypoint
que no sea `cli` caerá y se tratará como interactivo. Eso reintroduce la misma
clase de observación de sesiones automatizadas que este PR intenta evitar.

La regla más segura es:

- permitir solo `cli`
- tratar cualquier otro entrypoint explícito como automatizado
- mantener el fallback por defecto como `cli` cuando la variable no esté
  definida

Forma sugerida:

```bash
case "${CLAUDE_CODE_ENTRYPOINT:-cli}" in
  cli) ;;
  *) exit 0 ;;
esac
```

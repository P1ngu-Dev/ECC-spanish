# Paquete de ECC v2.0.0-rc.1 para socios, patrocinadores y charlas

Este paquete convierte la superficie de lanzamiento rc.1 en contenido listo para enviar a patrocinadores,
socios, conversaciones de consultoría, charlas en conferencias, reservas de podcasts y
anuncios comunitarios.

No es una acción de publicación. Úsalo después de que el registro de URL del release, la suite de video
y los puntos de control de publicación estén actualizados.

## Línea de posicionamiento

ECC 2.0 es el sistema operativo nativo del harness para trabajo agentic.

Usa esta versión corta en mensajes a socios y patrocinadores:

```text
ECC ofrece a los equipos una capa reutilizable para skills, hooks, rules, convenciones MCP,
gates de release y flujos de trabajo de operator across Claude Code, Codex, OpenCode,
Cursor, Gemini, Zed, GitHub Copilot y flujos de terminal únicamente.
```

## Socios objetivo

Prioriza socios que ya se benefician de una capa operativa agnóstica al harness:

- plataformas e IDEs de codificación con IA;
- herramientas alojadas de agentes y orquestación de flujos de trabajo;
- proveedores de code review, seguridad y cadena de suministro;
- proveedores de modelos e inferencia;
- organizadores de educación para desarrolladores, podcasts y conferencias;
- equipos que adoptan varios harnesses al mismo tiempo.

## Introducción de consultoría

```text
Estoy abierto a un número reducido de sprints de implementación de ECC 2.0 para equipos
que están estandarizando flujos de trabajo de codificación con IA.

El alcance útil normalmente es:

1. auditar la configuración actual del harness;
2. convertir flujos de trabajo repetidos en skills, hooks y rules de ECC;
3. añadir gates de release, seguridad y CI;
4. crear un bucle operativo de equipo que funcione en Claude Code, Codex, OpenCode,
   Cursor, Gemini, Zed, GitHub Copilot y flujos de terminal.

Esto no es consultoría genérica de IA. El resultado es un sistema operativo de harness
funcional que tu equipo puede seguir usando.
```

## Pitch para charla y podcast

Opciones de título:

- Building a Cross-Harness Operating System for AI Coding
- From Prompt Packs to Operator Systems
- What Breaks When Teams Adopt Too Many AI Coding Harnesses
- Security and Release Discipline for Agentic Coding Workflows

Pitch breve:

```text
ECC comenzó como una capa de flujo de trabajo de código abierto para Claude Code y ahora
avanza hacia un sistema operativo cross-harness para trabajo agentic.

La charla trata sobre los problemas prácticos que los equipos encuentran después de la luna de miel
inicial con la codificación con IA: prompts dispersos, configuración duplicada, gates de release
débiles, postura de seguridad frágil y ningún bucle operativo claro entre herramientas.

Puedo mostrar cómo ECC usa skills reutilizables, hooks, convenciones MCP, gates de release,
chequeos de seguridad estilo AgentShield y un shell de operator Hermes opcional para hacer que
el trabajo agentic sea más medible y portable.
```

## No enviar ni publicar si

- El registro de URLs del release todavía tiene enlaces obsoletos o de marcador de posición.
- `npm run release:video-suite -- --format json` no está en verde contra las raíces de video previstas.
- El prerelease de GitHub, el paquete npm, la ruta del plugin o el reclamo de facturación se describen como activos sin evidencia.
- El mensaje afirma que los pagos nativos están listos antes de que pase el readback de facturación de ECC Tools.
- El destinatario necesita una promesa personalizada que no está cubierta por `SPONSORS.md`, `SPONSORING.md` o un alcance de consultoría separado.
- El usuario no ha aprobado los mensajes outbound de patrocinio, socios, consultoría o medios.
